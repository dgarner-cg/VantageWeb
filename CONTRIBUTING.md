## 💡 Our Philosophy
When contributing, keep our core mission in mind:
1. **Safety over Noise:** Features should prioritize community well-being over "vanity" metrics and useless features with no vision for how they all work together to contribute to our mission and vision.
2. **Professionalism:** Our UI/UX should feel like a high-end management tool, our goal is to build tools and platforms that are regarded as the Apex the markets they compete for marketshar; we strive for perfection, without compromising our values, we want to be the 'Apple' of community management platforms. (Without the child labor)
3. **Scalability:** Code should be optimized to handle servers of any size. Our platform should be just as productive and helpful in a small community of close friends as it is to the Discord server of AAA Game Titles.

## 🛠️ How Can I Contribute?

### Reporting Bugs
* Check the **Issues** tab to see if the bug has already been reported.
* If not, open a new issue using the "Bug Report" template.
* Include steps to reproduce, your environment (Node version, OS), and expected vs. actual behavior.

### Suggesting Enhancements
* We love new ideas! Open an issue with the "Feature Request" tag.
* Explain *why* this feature helps community managers and how it fits into the "Vantage Point" vision.

### Code Contributions
1. **Fork** the repository and create your branch from `main`.
2. **Install dependencies** using `npm install`.
3. **Write clean code:** Follow the existing style and ensure your code is well-commented.
4. **Test:** Ensure your changes don't break existing bot commands or dashboard routes.
5. **Submit a PR:** Provide a clear description of the changes and link to any relevant issues.

## 🎨 Style Guidelines
- **Backend (Bot):** We prefer modular command handling and strict error catching to prevent bot downtime.
- **Frontend (Web):** Use Tailwind CSS for styling. Maintain a "Dark Mode" first aesthetic that aligns with Discord’s professional side.
- **Commits:** Use conventional commits (e.g., `feat: add onboarding workflow`, `fix: dashboard login loop`).

## 🤝 Code of Conduct
We are committed to providing a welcoming and inspiring community for all. By participating in this project, you agree to abide by our Code of Conduct:
* Be respectful of differing viewpoints and experiences.
* Gracefully accept constructive criticism.
* Focus on what is best for the community.
